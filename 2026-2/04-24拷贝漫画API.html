2. 公共请求头

所有 API 请求均携带以下 HTTP Headers（部分可配置开关）：

Header 值 说明
user-agent Mozilla/5.0 (Linux; Android 13; Pixel 7) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/141.0.0.0 Mobile Safari/537.36 Edg/141.0.0.0 可自定义

authorization Token {token} 或 Token（未登录） 认证令牌

accept application/json JSON 格式（可关闭）
accept-encoding gzip Gzip 压缩（可关闭）

5. 响应格式

5.1 通用响应结构

  • code 为 200 表示成功
  • 错误时 detail 字段包含错误描述

5.2 错误处理

  • HTTP 状态码 >= 400 时，从 errorStream 读取响应
  • 优先解析 JSON 中的 detail 字段作为错误信息
  • 非 JSON 响应视为异常

6. 代理与加密机制

6.1 API 代理

  • 匹配正则：^https://(api|www)\.(copymanga|mangacopy|copy-manga|copy20)\.\w+/api/
  • 需要配置 proxy_code

6.2 图片代理

  • 匹配正则：^https://[0-9a-z-]+\.mangafun[a-z]\.(xyz|fun)/

6.3 反向代理

  • 可配置自定义反向代理地址替代官方 API 主机
  • 配置后跳过 API 节点发现，直接使用反向代理

6.4 响应解密（libcomancry.so）

  • 通过 API 代理获取的响应需要用 libcomancry.so 原生库解密
  • 接口：decrypt(sourceDir: String, data: ByteArray): String?
  • sourceDir 为 APK 安装目录

6.5 网络增强（libcomandy.so）

7. API 节点容错

  1. 启动时调用 /api/v3/system/network2 获取可用节点列表
  2. 请求失败时自动移除失败节点，尝试下一个
  3. 所有节点失败时回退到配置的默认主机 api.2024manga.com
  4. 支持配置反向代理地址完全绕过节点发现
